## 🚀 ETL process for sales report (Project #1)

### 1. Data Quality Audit - Stellantis 🚗
**Problem:** Data fragmentation and lack of integrity within monthly sales reports, leading to unreliable business insights.

**Solution:** Developed a comprehensive SQL pipeline for duplicate and null detection, data standardization, and brand consistency cleansing.SQL files create views, those views also can be loaded for complete data cycle without affecting original database, keeping data integrity.
* [View GitHub code files](https://github.com/pegujosi/stellantis-sales-audit-engine/tree/main/scripts)
* [Technical documentation](https://github.com/pegujosi/stellantis-sales-audit-engine/blob/main/README.md)

### 2. Statistical Outlier Analysis & Data Distribution 📊
**Problem:** Standard averages were masking critical production deviations, leading to inaccurate performance metrics and overlooked anomalies.

**Solution:** Conducted a comprehensive statistical assessment using Python to justify outlier treatment. This included Normality Tests (Shapiro-Wilk) and Visual Analytics (Boxplots, QQ-Plots). I implemented Robust Z-Score and MAD (Median Absolute Deviation) to filter real operational outliers while preserving data integrity.
* [View Jupyter Notebook](https://github.com/pegujosi/stellantis-sales-audit-engine/tree/main/notebooks)

### 3. Sales Executive Dashboard 📈
**Problem:** Stakeholders lacked real-time visibility into shop-floor KPIs, leading to delayed decision-making and difficulty in identifying production and sales bottlenecks.

**Solution:** Developed an end-to-end Power BI dashboard (Tableau dashboard also available) by integrating SQL and Python. This solution centralizes manufacturing data to track critical metrics.
* [View Project Showcase](https://github.com/pegujosi/stellantis-sales-audit-engine)
